# 素材质量门槛：5-10-2-8 规则

> 适用 skill：competitor-analysis、intel-collector
> 目的：防止以量代质的素材堆积，只收录真正有信息价值的竞品素材。

---

## 5-10-2-8 规则定义

| 数字 | 含义 |
|------|------|
| **5** 轮搜索 | 每个目标功能 / 模块至少执行 5 轮独立搜索（不同关键词 / 不同平台），确保覆盖面 |
| **10** 个候选 | 每轮结束后汇总 ≥ 10 个候选素材（截图 / 页面 / 文案片段） |
| **2** 个入选 | 从 10 个候选中只选最具代表性的 2 个进入分析报告 |
| **8** 分门槛 | 每个入选素材必须 ≥ 8/10 分（评分标准见下） |

---

## 素材评分标准（10 分制）

| 维度 | 权重 | 说明 |
|------|------|------|
| **信息密度** | 30% | 能否从这个素材直接提取出业务规则 / 流程节点 / 数据字段 |
| **时效性** | 25% | 素材是否来自近 6 个月内的功能版本（截图含日期 / 版本号更佳）|
| **代表性** | 25% | 是否覆盖目标竞品的核心路径（而非边缘 / 异常态页面）|
| **可引用性** | 20% | 是否可直接被 PRD / 交互大图引用说明（非模糊截图 / 非纯文字描述）|

**计分说明**：四维度加权平均，满足 ≥ 8/10 才入选；不足 8 分的素材进入「备选库」不进报告。

---

## 采集流程

```
1. 确认采集目标（功能模块 + 目标平台列表）
2. 执行 5 轮搜索，记录候选素材列表（≥10 个）
3. 对每个候选素材按 4 维评分
4. 选取评分最高的 2 个入选
5. 不达标素材 → stderr 记录，不写入报告
```

---

## 质量自检 log 格式

采集脚本在每个素材评分后输出：

```
[ASSET-RUBRIC] {素材标识}
  信息密度: {分}/10 | 时效性: {分}/10 | 代表性: {分}/10 | 可引用性: {分}/10
  加权总分: {分}/10 — {'✅ 入选' if 分>=8 else '⚠️ 备选，不进报告'}
```

总分 < 8 的素材输出到 **stderr**（而非 stdout），不影响主流程，不进报告正文。

---

## 注意事项

- 5 轮搜索中允许跨平台（App Store、网页端、新媒体报道、帮助中心），但不同来源同一功能页面算 1 个候选
- 「素材」的最小单位是：一个完整的流程截图 / 一段可引用的规则文案 / 一个可分析的 UI 状态
- 备选库保存在 `projects/{项目名}/inputs/competitor-raw/`，不进 deliverables
