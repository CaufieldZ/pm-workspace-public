# 内容纪律（Content Slop 黑名单）

> 适用：prd skill、data-report skill。每次产出前必须过一遍此清单。

---

## 禁止列表

### 禁 1：编造数据（Data Slop）

```
❌ "10,000+ 用户已参与"
❌ "转化率提升 30%"（没有真实来源）
❌ "99.9% 可用性"（无测量数据）
❌ 带图标 + 数字 + 词组的 metric card 装饰
```

**规则**：没有真实数据来源（神策拉取 / 产品实测 / 运营提供）的数字，统一用：

```markdown
[数据待填：{指标名称}]
```

### 禁 2：编造用户评价 / 引言（Quote Slop）

```
❌ "用户反馈说这个功能很好用"（未经确认）
❌ 名人名言装饰页面
```

没有真实 quote → 留 `[用户引言待填]` 或不放。

### 禁 3：Filler Content（填充废话）

判断一段内容是否值得保留的标准：

> **去掉这段，文档变差了吗？不会变差 → 删掉。**

典型 filler 症状：

- 「本功能旨在提升用户体验，改善用户满意度……」（泛泛目标）
- 「综上所述，本次需求的重要性不言而喻」（总结废话）
- 每个章节开头的一段背景介绍（可以直接从需求点开始）
- 为了「页面不空」而补充的内容

### 禁 4：单方面增加内容（Ask Before Adding）

> 想加一段 / 一页 / 一个 section → 先问用户，不要自行加。

原因：用户比模型更了解读者需求；加内容有成本，用户可能不需要。

## Scale 规范（PRD → docx 转换时强制）

| 元素 | 规格 |
|------|------|
| 正文 | **11pt**（最小 10pt，最大 12pt）|
| 标题 1 | **24pt** |
| 标题 2 | **18pt** |
| 标题 3 | **14pt** |
| Caption / 注释 | **8-9pt** |
| 行高 | 正文 1.5 / 标题 1.2 |
| 页边距 | 上下 2.54cm / 左右 3.17cm（Word 默认）|

## PRD 特有：业务规则完备性检查

每个功能点产出后，自检：

- [ ] 是否说明了触发条件？
- [ ] 是否说明了成功态 + 失败态？
- [ ] 是否说明了频控 / 门槛 / 白名单等数值限制？
- [ ] 数值是否来自 context.md 第 6 章？有无「TBD / 待确认」未填项？

有「待确认」字段 → 在 PRD 中用 `**[待确认]**` 显式标记，不静默留空。

---

## 数据报告特有：数值来源强制标注

每个数据点必须注明来源：

```markdown
- 次留率：42.3%（神策 SQL：event_user_next_day_retention，2026-04-01~04-07）
- DAU：12,847（有数 BI 日报 2026-04-07）
```

来源缺失时写：`[数据待核实，来源：{系统名}]`

禁止写「约 X%」「大约 X」等模糊表达（除非真实数据本身是近似值并说明理由）。
