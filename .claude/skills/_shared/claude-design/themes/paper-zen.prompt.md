## 设计语言研究任务：「学院派教研文档」美学底座

## 任务

我提供一套已落地的 HTML 设计案例（5 张幼教研修类信息图），请你深度拆解它的设计语言，沉淀成一套**可复用的"美学底座"**，输出为我未来设计产出物（PRD 配图、方案架构图、研究报告、教研文档、知识卡片等）时可以直接调用的设计 token + 构图模板 + 决策原则。

请不要只做表面分析（"配色和谐、字体优雅"），要拆到能复刻的颗粒度——具体的色值、字号、间距、骨架、决策依据。

## 这套设计语言的核心定位

**不是**：国潮古风、互联网信息图、SaaS 落地页、幼教卡通风、Material Design、Apple 风格
**是**：东方质感的西方学院派 / 出版物美感 / 研究报告封面感

参考坐标系：
- 大学出版社年报、博物馆图录、学术期刊封面
- 国内出版物：《读库》《单读》《信睿周报》《三联生活周刊》深度报道版面
- 国际参考：FT Weekend、纽约时报特别报道、麦肯锡 / BCG 报告封面、哈佛商业评论
- 共通点：**怕的不是"不漂亮"，怕的是"不像出版物 / 不专业 / 不耐看"**

主动避开的反面参考：
- emoji 图标（太互联网）
- 渐变光效、霓虹色、玻璃拟态（炫技但不耐看）
- 纯黑大字（视觉重量过死）
- 重 box-shadow 卡片（像 SaaS 落地页）
- 卡通插画、手写字体（除非目标受众是儿童 / 家长）

## 设计 Token（请整理成可直接复制的 CSS 变量块）

### 配色

**底色系（纸张感）**
- `--bg: #FAF6EC` 主背景，仿米黄羊皮纸
- `--paper: #FFFDF7` 卡片底，比主底略亮
- `--line: #D9CFB8` 浅分隔线，米色调
- 关键判断：**底色绝不用纯白**，纯白是 Web 默认色，没有"质地感"

**文字系**
- `--ink: #1F1B16` 主文字，深棕黑（不用纯黑 #000，纯黑过硬）
- `--ink-soft: #5C5447` 次级文字，灰棕

**主色系（权威感）**
- `--accent: #1B3A2F` 深墨绿，主标题 / 主分隔线 / 强调用
- `--gold: #A8804A` 金棕，副强调 / 装饰线 / 点睛色

**多层级配色（5 阶情绪光谱）**
- `--s1: #D89A4B` 琥珀（暖起点）+ `--s1-soft: #F4E3C8`
- `--s2: #789B6C` 生长绿 + `--s2-soft: #DCE7D4`
- `--s3: #3F7A95` 湖蓝 + `--s3-soft: #CFE0E8`
- `--s4: #B85540` 陶土红 + `--s4-soft: #EFD3CB`
- `--s5: #4F3A65` 深紫 + `--s5-soft: #DDD2E5`

色相逻辑：**色温从暖→冷→深**，暗合"感性→理性"的认知温度递进。每个主色都配一个 -soft 浅版用作背景或标签底。

### 字体

中文：
- 主体：`"Noto Sans CJK SC", "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`
- 标题：`"Noto Serif CJK SC", "Noto Serif SC", "Songti SC", serif`

搭配规则：**标题用宋体衬线 + 正文用黑体无衬线**（出版物经典三件套）。所有 H1/H2/重点强调用宋体，所有正文/标签/数据用黑体。这是国内出版物的标准搭配。

### 字号阶梯（信息图场景，1600-1800px 画布）

- H1 主标题：46-56px，weight 900
- 副标题：18-21px，ink-soft 色
- 区块标题（H2）：24-30px，weight 900
- 卡片大标题：28-32px，weight 900，宋体
- 强调动作短语：21px，weight 700
- 正文：18-21px，weight 500
- 标签 / pill：15-17px，weight 700
- 元信息 / 元数据：13-15px

**关键判断：信息图字号要比 Web 文档大 30-50%**——因为最终大概率会被截图、缩放、嵌入 Word/PPT。"看起来略大"就是"刚刚好"。

### 间距与圆角

- 圆角：2-3px（轻微圆角，避免完全直角的"古板"和大圆角的"互联网"）
- 卡片 padding：22-26px
- gap：14-18px
- 阴影：`0 14px 24px -16px rgba(31,27,22,.22)` 极轻阴影，几乎不可见，只在视网膜屏才显影
- 边框：`1.5px solid var(--line)` + 顶部 `5-7px solid var(--c)` 强调色横条

## 构图骨架库（可调用的版面模板）

请整理成几个命名清晰的骨架，每个说明：用什么场景、骨架结构、何时用。

### 骨架 A：报头 + 主轴 + 内容
- 顶部：左标题 + 右元信息（用左侧 1.5px 竖线分隔），底部 2px 主色横线 + 7px 金棕短装饰线
- 中部：起点标签—虚线/渐变轴—终点标签
- 下部：内容区
- 适用：路径图、流程图、进阶图

### 骨架 B：3×3 矩阵
- 横轴：能力梯度（基础→进阶→挑战）
- 纵轴：维度（材料/提问/指导）
- 顶行用主色填底反白字 + 大圆形罗马数字徽章
- 适用：分层支持图、能力模型图、对比矩阵

### 骨架 C：阶梯式卡片
- 5 张卡片用 `margin-top: 64/48/32/16/0px` 实现楼梯感
- 卡片间用菱形连接点（CSS 旋转方块 + 阴影）
- 适用：进阶路径、成长阶段、roadmap

### 骨架 D：底部彩色渐变条带 + 三/五栏小结
- 浅米色块 `#F2EAD3` + 顶部 5px 渐变条（横跨所有阶段色）
- 内部 3-5 栏并列，每栏：色块标签 + 宋体小标题 + 黑体描述
- 适用：原则/总结/结论区

## 视觉小动作清单（点睛细节）

请整理几个"小动作"——单独看不起眼但累加起来决定专业度的细节：

1. 主标题里挑一个关键词加**浅色高亮块**做"荧光笔"效果（用 linear-gradient 实现 60% 处分界）
2. 报头底部 2px 主色横线之外，再叠一条 120px 短**金棕装饰线**做点睛
3. 圆形数字徽章：彩色描边 + 浅色填底 + 主色文字（不是实心填充）
4. 分隔线用**虚线**而不是实线（出版物语言）
5. 主轴用 `repeating-linear-gradient` 做**断续虚线**，末尾加 CSS 三角箭头
6. 标签 pill 用 `--c-soft` 浅版底色配 `--c` 文字（同色系深浅对比）
7. 引号用全角"" 不用半角 ""，破折号用 — 不用 -

## 决策原则（最重要的部分）

请整理成"什么时候用 / 什么时候不用"的判断框架，至少包括：

1. **受众判断**：这套语言适合什么人看？教研员、园长、研究者、专业读者。**不适合**儿童、家长、互联网年轻用户、需要快速决策的高管（高管要更直白、更数据驱动的版式）。

2. **场合判断**：适合公开课说课、园本研修、研究报告、深度方案文档、PRD 配图、知识卡片。**不适合**家长会通知（太冷）、用户增长素材（太慢）、儿童学习材料（太成人）、紧急通知（太装饰）。

3. **内容判断**：适合**结构化思考的产物**——分层、分阶段、对比、进阶、原理。**不适合**——纯数据 dashboard（要更冷峻的灰白色板）、营销卖点页（要更鲜活）、操作指南（要更直给）。

4. **替代方案**：如果不适合这套，应该走什么风格？给出 3-4 套替代美学底座的简要描述（深色科技风 / 暖色家长风 / 国潮古风 / 极简数据风），方便我快速换轨。

## 输出要求
请你以"设计师做项目复盘"的视角而不是"分析师写报告"的视角来做这件事，目标是产出一份**未来真的能拿来用**的工作底座，不是一份漂亮但放着吃灰的分析。
