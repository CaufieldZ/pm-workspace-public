## 填充规范

### 模块拆解

- 每个页面的模块编号从 M1 开始，文档内全局唯一（如 Scene A 的 M1 和 Scene B 的 M1 不同，但同场景内不重复）
- 类型必须是以下之一：导航 / 内容 / 操作 / 反馈 / 筛选 / 分隔
- 「交互行为」描述用户可以做什么，「操作结果」描述界面发生什么变化

### 状态矩阵

- 状态列从 PRD 的状态定义继承，不自创状态
- 每个单元格必须包含：展示内容 + 视觉样式（至少描述颜色和可操作性）
- 模块在某状态下不存在 → 写「隐藏」
- 模块在某状态下无变化 → 写「同默认态」

### 交互流

- 步骤编号连续（1, 2, 3...），不跳号
- 每步包含：页面展示 + 视觉反馈 + 页面变化
- 主路径只覆盖最短成功路径，异常路径单独在 3.x.4 描述
- 不写 Mermaid / 流程图，纯文字 + 步骤编号

### 异常路径

- 每个页面至少覆盖：Loading / 空态 / 网络错误
- 视觉表现必须具体：不说「显示错误提示」，说「顶部红色 Toast，文案"网络异常，请重试"，2 秒后消失，右侧有重试按钮」
- 恢复方式必须明确：自动恢复 / 用户操作恢复 / 页面刷新

### 术语表

- 文档内首次出现的术语必须有定义
- 「视觉对应」列帮助设计师 AI 快速定位 UI 元素
- 来源优先级：scene-list.md → context.md → PRD 内定义

### 跨页面交互

- 跳转表描述页面间的导航关系
- 过渡动画描述视觉表现（右滑入、底部弹出、淡入淡出等）
- 共享模块表只列有差异的复用模块

### 配套说明

- 如有可交互原型 HTML，写入文件路径
- 如无原型，写「原型待生成」
- 不强制要求原型存在

## 切分多份文档时的规范

当按页面/模块拆分为多份独立文档时，每份必须：

1. **独立的术语表**：只包含本页面用到的术语
2. **独立的状态矩阵**：本页面涉及的模块和状态
3. **独立的异常覆盖**：本页面的异常路径
4. **跨文档引用用自然语言**：如「从首页活动卡片点击进入（见「首页」页面结构 Scene A M2）」
5. **各自通过自检**：每份文档独立执行自检清单

文件命名：`pspec-{项目简称}-{页面名}-v{N}.md`
