会后补录：拉最近一条 A1 录音的闪记纪要，结构化后归档到对应项目目录。

参数：$ARGUMENTS 可传入项目名（如 `htx-queen`），不传则自动匹配。

跳过录音控制环节，直接从 Step 5 开始。

## 执行步骤

### Step 1：获取最新录音

```bash
cd .claude/skills/meeting-autopilot
python3 scripts/call_mcp.py call "$MCP_A1_URL" get_dingtalk_a1_file_list --params '{"pageSize": 5}'
```

列出最近 5 条录音，让用户确认要整理哪一条。如果只说了「整理纪要」不带上下文，默认取最新一条。

检查 `minutes.state`：
- `state == 2` → 转写完成，继续
- 其他 → 告知用户「这条录音的闪记还没生成完，稍后再试」

### Step 2：拉纪要内容

闪记完成后，钉钉会自动生成文档（文档名格式 `纪要_{会议标题}`）。

```bash
python3 scripts/call_mcp.py call "$MCP_DOC_URL" search_documents --params '{"keyword": "<会议标题>"}'
python3 scripts/call_mcp.py call "$MCP_DOC_URL" get_document_content --params '{"nodeId": "<搜索到的 nodeId>"}'
```

### Step 3：匹配项目

如果 $ARGUMENTS 非空，直接用该项目名。否则：

1. `ls projects/` 获取全部项目目录名
2. 将会议标题分词，与目录名做包含匹配
3. 匹配不上 → 用 AskUserQuestion 问用户「这场会归到哪个项目？」，列出项目 + misc 选项
4. 用户选 misc → 归到 `projects/_misc/inputs/`

### Step 4：生成结构化纪要

基于闪记内容，按以下模板整理：

```markdown
# 会议纪要：<会议名称>

**日期**：YYYY-MM-DD HH:MM - HH:MM
**参会人**：<姓名列表>
**录音文件**：<闪记 URL>

## 核心结论
- <结论1>
- <结论2>

## 讨论要点
### 议题一：<议题>
- 讨论内容摘要

## 行动项
| # | 任务 | 负责人 | 截止日期 |
|---|------|--------|----------|
| 1 | <具体任务> | <姓名> | <日期> |

## 下次会议
- 时间：<待定/具体时间>
```

### Step 5：存档

写入 `projects/{项目名}/inputs/meeting-YYYY-MM-DD.md`。
同一天多场：`meeting-YYYY-MM-DD-2.md`。
`_misc`：`meeting-YYYY-MM-DD-{主题简写}.md`。

### Step 6：context.md 变更确认

提取决策结论 + 行动项，先输出摘要给用户过目。用户确认后追加到 context.md 第 7/9 章。用户说"不更新"则跳过。

### Step 7（可选）：钉钉待办 + 机器人推送

询问用户是否需要创建钉钉待办或推送纪要。
