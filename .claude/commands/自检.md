对 pm-workspace 执行全局自检，逐项用 bash 命令检查，直接输出结果，不要探索目录。

按以下顺序检查：

## 1. 核心配置文件

检查以下文件是否存在，输出行数和字符数：
- CLAUDE.md
- .claude/rules/pm-workflow.md
- .claude/rules/soul.md
- .claude/chat-templates/context-template.md

## 2. Skill 完整性

遍历 .claude/skills/*/，每个目录必须有 SKILL.md。

## 3. 自定义命令

ls .claude/commands/ 列出所有命令。

## 4. 生成脚本

检查以下文件是否存在：
- .claude/skills/interaction-map/references/gen_imap_skeleton.py
- .claude/skills/prd/references/gen_prd_base.py
- .claude/skills/prototype/references/gen_proto_skeleton.py

## 5. Token 负载

计算 CLAUDE.md + .claude/rules/*.md 的总字符数和估算 token 数（chars/4），目标 < 5000t。

## 6. 组件库

检查 .claude/skills/interaction-map/references/components-cheatsheet.md 的组件数（grep -c "^####"），目标 >= 20。

## 7. 项目目录

遍历 projects/*/，每个项目检查 context.md、scene-list.md、scripts/、deliverables/ 是否存在。

## 8. Git 状态

运行 git status --short，报告未提交的变更。

全部用 bash 命令执行，一次性跑完，最后输出汇总：✅ 数量 / ❌ 数量 / ⚠️ 数量。有 ❌ 的列出修复建议。
