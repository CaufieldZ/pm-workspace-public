会议录音全流程：日历感知 → A1 录音控制 → 闪记转写 → 结构化纪要 → 归档到项目目录。

参数：$ARGUMENTS 可传入项目名（如 `htx-queen`），不传则自动匹配。

## 执行步骤

### Step 1：查今日日历

优先尝试 Outlook MCP（`list-events`，按今日日期过滤），若 Outlook 未授权或调用失败，降级用钉钉日历：

```bash
cd .claude/skills/meeting-autopilot
python3 scripts/call_mcp.py list "$MCP_CALENDAR_URL"
python3 scripts/call_mcp.py call "$MCP_CALENDAR_URL" <查询日程工具名> --params '{"date": "<今日日期>"}'
```

输出今日日程概览表（时间、会议名、参会人、状态）。

### Step 2：匹配项目

如果 $ARGUMENTS 非空，直接用该项目名。否则：

1. `ls projects/` 获取全部项目目录名
2. 将会议标题分词，与目录名做包含匹配：
   - "一姐" / "queen" → htx-queen
   - "流动性" / "liquidity" / "LT" → LT-HTX-Liquidity
   - "活动中心" / "activity" → htx-activity-center
   - 以此类推，目录名本身就是关键词
3. 匹配不上 → 用 AskUserQuestion 问用户「这场会归到哪个项目？」，列出项目 + misc 选项
4. 用户选 misc → 归到 `projects/_misc/inputs/`，文件名带主题简写

### Step 3：A1 录音控制

```bash
cd .claude/skills/meeting-autopilot
# 查设备
python3 scripts/call_mcp.py call "$MCP_A1_URL" get_dingtalk_a1_device_info --params '{}'
# 开始录音
python3 scripts/call_mcp.py call "$MCP_A1_URL" send_dingtalk_a1_task_command --params '{"action": "start"}'
```

告知用户「A1 已开始录音，会议结束后告诉我"停止录音"」。

用户说"停止录音"时：
```bash
python3 scripts/call_mcp.py call "$MCP_A1_URL" send_dingtalk_a1_task_command --params '{"action": "stop"}'
```

用户说"暂停/恢复"时对应 pause/resume。

### Step 4：等待闪记生成

录音停止后，闪记转写需要时间。轮询检查：

```bash
python3 scripts/call_mcp.py call "$MCP_A1_URL" get_dingtalk_a1_file_list --params '{"pageSize": 1}'
```

检查返回的最新录音 `minutes.state`：
- `state == 2` → 转写完成，继续
- 其他 → 等待 30s 后重试，最多等 5 分钟
- 超时 → 告知用户「闪记还在生成中，稍后可用 /整理纪要 手动触发」

### Step 5：拉纪要内容

闪记完成后，钉钉会自动生成文档（文档名格式 `纪要_{会议标题}`）。通过钉钉文档 MCP 获取：

```bash
# 按会议标题搜索文档
python3 scripts/call_mcp.py call "$MCP_DOC_URL" search_documents --params '{"keyword": "<会议标题>"}'
# 获取文档 Markdown 内容
python3 scripts/call_mcp.py call "$MCP_DOC_URL" get_document_content --params '{"nodeId": "<搜索到的 nodeId>"}'
```

### Step 6：生成结构化纪要

基于闪记内容，按以下模板整理（保留闪记的核心信息，精简格式）：

```markdown
# 会议纪要：<会议名称>

**日期**：YYYY-MM-DD HH:MM - HH:MM
**参会人**：<姓名列表>
**录音文件**：<闪记 URL>

## 核心结论
- <结论1>
- <结论2>

## 讨论要点
### 议题一：<议题>
- 讨论内容摘要

## 行动项
| # | 任务 | 负责人 | 截止日期 |
|---|------|--------|----------|
| 1 | <具体任务> | <姓名> | <日期> |

## 下次会议
- 时间：<待定/具体时间>
```

### Step 7：存档

写入 `projects/{项目名}/inputs/meeting-YYYY-MM-DD.md`。
如果同一天有多场会议，文件名加序号：`meeting-YYYY-MM-DD-2.md`。
`_misc` 项目的文件名带主题：`meeting-YYYY-MM-DD-{主题简写}.md`。

### Step 8：context.md 变更确认

从纪要中提取：
- **决策结论**（对应 context.md 第 7 章）
- **行动项 / 待办变更**（对应 context.md 第 9 章）

先输出变更摘要给用户过目，格式：

```
以下内容将追加到 context.md：

【第 7 章 · 关键决策】
- <决策1>
- <决策2>

【第 9 章 · 待办 & 下一步】
- <行动项1>

确认追加？(y/n/编辑)
```

用户确认后才写入。用户说"不更新"则跳过。

### Step 9（可选）：钉钉待办 + 机器人推送

询问用户是否需要：
- 为行动项创建钉钉待办（调 `$MCP_TODO_URL`）
- 通过机器人推送纪要摘要（调 `$MCP_BOT_URL`）
